## Quick reference

```bash
uv pip install -e ".[dev]"   # install (requires Python >=3.12)
uv run ruff check .          # lint
uv run mypy src/             # typecheck (strict mode)
uv run pytest tests/         # 194 tests, ~15s
```

`pyproject.toml` lists `pytest-xdist` but `-n auto` only works after the full `.[dev]` install. Safe to omit it — the suite is fast enough single-threaded.

## Project structure

This repo **consumes** JSONL instances from a separate SWE-benchify repo and runs cost-tiered model sweeps against them. It never mines PRs or builds instances.

```
src/swe_routing_eval/
  ingest.py        # Pydantic SWEbenchInstance, JSONL loader
  vertex.py        # VertexConfig from env vars, tier→model_id
  cost.py          # PriceTable, cascade_point()
  budget.py        # dry_run_estimate()
  store.py         # RunRecord, FileStore (SQLite), Store ABC
  grading.py       # Grader Protocol, safe_grade pipeline, anti-reward-hacking
  scaffold.py      # Fixed agent scaffold (Anthropic Vertex + Cursor CLI)
  orchestrator.py  # Sweep scheduler with resume, budget guard, circuit breaker
  stats.py         # bootstrap_ci, mcnemar_test, connor_power
  frontier.py      # Pareto frontier, memo renderer, plot
  cursor_usage.py  # Cursor dashboard API client for GPT cost tracking
  parser.py        # GoJSONParser Protocol stub (blocked on SWE-benchify issue #2)

scripts/           # CLI entry points — not importable modules
  eval_sweep.py    # Main sweep runner (--dry-run or full)
  analyze_runs.py  # Produce memo + plot from runs.db
  regrade.py       # Re-grade stored patches without re-running inference
  spot_audit.py    # Human review harness (generate/record/summary)
  export_dashboard_data.py  # runs.db → docs/data.json
  m0_coverage_check.py      # Gold-patch validation gate

config/prices.json # Vertex pricing at RH committed-use rates
docs/              # GitHub Pages dashboard + design docs
```

## Architecture gotchas

- **Scripts use `importlib.util` to load other scripts.** Tests for `m0_coverage_check.py`, `eval_sweep.py`, `analyze_runs.py`, and `spot_audit.py` import `main()` via `spec_from_file_location`, not normal imports. Don't move scripts or change their `main()` signatures without updating the corresponding tests.

- **Two scaffold implementations.** `Scaffold` (Anthropic Vertex API) and `CLIScaffold` (Cursor `agent` CLI). The orchestrator picks based on whether the tier is in `VERTEX_TIERS` (`vertex.py:16`). Non-Vertex tiers route to `CLIScaffold`.

- **`product` field is always overwritten.** `SWEbenchInstance._product_from_repo` (`ingest.py:67-69`) sets `product = self.repo` regardless of input. Downstream code (stats, frontier, dashboard) groups by `product` as the segment key — it's effectively the `owner/repo` string.

- **`parser.py` is a stub.** `get_parser()` raises `ImportError`. The `GoJSONParserProtocol` exists only to define the contract for when SWE-benchify exposes the parser (issue #2). Don't try to call it.

- **Store has schema migrations.** `FileStore.__init__` (`store.py:100-106`) adds `grader_error` and `cli_scaffold` columns to older DBs via `ALTER TABLE`. New columns need the same pattern.

## Grading pipeline

`safe_grade()` (`grading.py:277-293`) is the full pipeline — always use it, never call `grader.grade()` directly:

1. Empty patch → `resolved=False`
2. `touches_test_files()` — rejects patches modifying `*_test.go` or `testdata/`
3. `grader.grade()` — Docker-based execution
4. `apply_quarantine()` — filters quarantined tests; etcd restricts F2P to unit packages

The grader has two implementations: `SwebenchifyGrader` (importable library, default) and `SubprocessGrader` (calls `swe-grade` binary). The `swebenchify` package is not in `pyproject.toml` deps — it must be installed separately from the SWE-benchify repo.

## Testing patterns

- No `conftest.py` — each test file constructs its own fixtures inline.
- Subprocess calls (grader, Anthropic API, Cursor CLI) are mocked with `unittest.mock.patch`.
- `tmp_path` for all SQLite store tests.
- `monkeypatch` for Vertex env vars.
- Script tests import `main()` and call it with argv lists, not via subprocess.

## Style and CI

- **ruff:** `line-length = 100`, selects `["E", "F", "I"]`
- **mypy:** `strict = true`, `ignore_missing_imports = true`
- CI runs `ruff check .` → `mypy src/` → `pytest tests/ -n auto` (lint → typecheck → test)

## Data files

- `runs.db` — SQLite run store, checked in with real eval data. Primary key: `(model_id, instance_id, attempt_idx)`.
- `instances-go.jsonl` — checked in. `instances.jsonl` is referenced but not in the repo.
- `docs/data.json` — generated by `export_dashboard_data.py`, committed for GitHub Pages.
- `config/prices.json` — real RH committed-use rates. Has a `gpt-*` wildcard fallback in `PriceTable._lookup` (`cost.py:33-34`) for GPT model variants not explicitly listed.
